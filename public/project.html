Meticulous– Chronos alpha 1

Overview

Team Chronos has been approached by a medium sized business to create a prototype organisation app for their employees; the company was interested in the potential of the project and would like to fund the development of an alpha version. As development continues more features and better UI/UX will be implemented so as to achieve the long term goals of the project.

Motivation

There is a market opportunity for a brilliant organisation app; while most only offer a simple calendar and to do list as their final product, this will be Chronos’ Alpha. While the original vision is highly functional,polished and complex, the alpha must be simple, effective and functional.

Therefore, we must create a solid foundation for a fully featured organisation app, the code must be written efficiently and utilise minimal machine resources whilst also being extensible. A lot of thought must be put in to the development so as to simplify expansion and polishing of the functionality and user experience.

Description

The alpha application requires two basic features – a calendar and a to do list.The earliest versions of the application will run completely as a command line interface on the desktop, from there modules can be added and expanded without the risk of encountering bugs produced by the web client.

Development of the alpha command line version will stop with an adequately featured calendar and to do listA web client will be produced using HTML5 and CSS and interact with the Python scripts through the use of an API which manipulates the DOM, creating a dynamic GUI.Development of the alpha web client version will stop with a functional and effective GUI and functional integration with the server side scriptsFurther polishing will be required to create a high-quality product, but through creating a solid foundation, further development will be relatively simple as) additional python modules can be added piece by piece and) the user experience can be developed on top of an existing codebase rather than assumptions of functionality.

Project Research

Who else will want this app? Is there a market similar to the users of Tiimo? Advertising – what channels do we direct our resources to? Costs – servers, development, marketing, leadership, tax, app store fees, royalties (logo, font), licenses (APIs) Development time – time until profit, timeline of versions, when is first sellable version available?

Tooling

The applet will be written in python for parallel development on desktop and web clients – the platform will be accessible via a desktop running a python client OR a modern web browser equipped with JavaScript/HTML 5/CSS 3.

The web app will run server-side using node.js to run python modules and display the data and UI to the user. Data will be saved locally OR to a web server and a login system will be used to sync user accounts and data together.Python development will take place in a python development environment i.e.VS code or Pycharm.

Web development will take place in a web development environment ie DreamWeaver or VS code. A possible tool which can bridge the python to web development is TRANSCRYPT, which converts the python code to JS code for the scripts to interact with the DOM.A web browser to view the app i.e.chrome.
